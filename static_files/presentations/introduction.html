PROLOGUE · 開場

抽象化、ADT 與這門課的地圖 cppds §1.1–1.6

寫程式 = 把問題的資料與過程編碼進電腦。 抽象資料型別（ADT）只告訴你「能做什麼操作」（介面），把「怎麼做到」（實作）藏起來—— 同一個 ADT 可以有很多實作，效能各不相同。資料結構 = ADT 的實作。

為什麼要學？ ① 大問題靠抽象化管理複雜度；② 同一問題的不同解法差距可以是「秒 vs 年」（第 2 章起量化）； ③ 這些結構是所有系統的積木：OS、資料庫、編譯器、AI 框架。
C++ 的定位 C++ 讓我們貼著記憶體學資料結構：指標、手動配置（new/delete）、值語意—— Python 幫你藏起來的細節，這裡全部攤開。懂了底層，回頭用什麼語言都更強。
PART 01 · 原子型別

內建原子型別：每個變數都是「固定大小的格子」 cppds §1.8

C++ 是靜態型別語言：宣告變數就固定了型別與大小。 int、double、char、bool 是最基本的「原子」。

›點型別卡看它的大小與取值範圍。
與 Python 的分野
Python 的 int 無上限、變數可隨時換型別； C++ 的 int 通常 4 bytes、溢位不會報錯（21 億再 +1 變負的！）。大小固定正是陣列 O(1) 定址的前提。
整數除法陷阱
9 / 5  → 1  // int ÷ int = int
9.0 / 5 → 1.8
9 % 5  → 4
PART 02 · 指標

指標：存「位址」的變數 cppds §1.8（Pointers）

指標是整門課的關鍵道具（第 4 章串列、第 8 章樹全靠它）。 兩個運算子：& 取位址、* 解參考（沿位址取值）。

›按步驟按鈕，看變數格與指標格的變化。
程式對照 CODE
int varN = 100; // 一般變數：住著值 int *ptrN = &varN; // 指標：住著位址 cout << ptrN; // 印出位址（如 0x7ffe...） cout << *ptrN; // 解參考：順著位址取值 → 100 *ptrN = 50; // 透過指標改 varN ptrN = NULL; // 指向「無」；解參考 NULL = 當機

PART 04 · 函式

傳值 vs 傳參考：影本還是本人？ cppds §1.10

C++ 預設傳值（複製一份）；加 & 變傳參考（操作原變數）。 大型容器傳值會整包複製——所以慣例寫 const vector<int>& v（借看不借改）。

›比較三種傳法對呼叫端變數 a 的影響。
三種傳法 CODE
void byValue(int x)  { x = 99; } // 拿到影本 void byRef(int &x) { x = 99; } // 拿到本人 void byPtr(int *x) { *x = 99; } // 拿到地址 int a = 1; byValue(a); // a 仍是 1 byRef(a); // a 變 99 byPtr(&a); // a 變 99
陣列參數的特例
C 風格陣列當參數會退化成指標—— 函式裡改 a[i] 就是改原陣列，且 sizeof 資訊遺失。vector 沒這個坑。
PART 05 · 類別

自訂型別：Fraction 與運算子多載 cppds §1.11

類別 = 狀態（成員變數）＋行為（成員函式）。運算子多載讓 f1 + f2、f1 == f2 對自訂型別也成立——這是把 Fraction 變成「一等公民」的關鍵。 建構子裡先約分（gcd），確保 3/6 一進門就變 1/2。

+ =?
›輸入兩個分數（a/b 格式），選運算。
Fraction CODE
class Fraction { // dscpp/fraction.hpp private: int num, den; // 狀態（private：封裝） public: Fraction(int t, int b) { num = t/gcd(t,b); den = b/gcd(t,b); } Fraction operator+(Fraction o) { return Fraction(num*o.den + o.num*den, den*o.den); } bool operator==(Fraction o) { return num*o.den == o.num*den; } };
封裝的意義
num/den 設成 private：外界只能走公開介面， 「永遠是最簡分數」這個不變量就有人守門。

類別層級 CODE
class LogicGate { ... getOutput() 呼叫 performGateLogic() ... }; class BinaryGate : public LogicGate { ... 兩個輸入腳位 ... }; class AndGate : public BinaryGate { int performGateLogic() { return getPinA() && getPinB(); } }; // Connector 把 fromGate 的輸出接到 toGate 的輸入 → 組合電路
多型在哪？
父類的 getOutput() 呼叫 performGateLogic()，但實際執行的是子類的版本 （C++ 用 virtual 實現）——同一介面、各自行為。
