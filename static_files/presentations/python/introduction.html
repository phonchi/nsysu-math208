PROLOGUE · 開場

演算法、抽象化，還有這門課要帶你去哪 pythonds §1.1–1.6

面對一個問題，資訊科學家的目標是寫出演算法：一份一步一步的指令清單， 能解決這個問題的任何一個實例。講義給了正式定義，四個條件缺一不可：

① 定義良好
指令是有序的集合，每一步都明確。
② 步驟不含糊
說「把兩個整數相加」，就得先講清楚什麼是整數、什麼是加。
③ 產生結果
回傳資料，或造成某種效果（例如印出東西）。
④ 有限時間內停止
跑不完的程序不算演算法。存在演算法可解的問題，我們說它是「可計算的」（computable）。

資訊科學同時也是研究抽象化的學問。開車不需要懂引擎， 你只用方向盤、油門、煞車這組「介面」。寫程式也一樣：import math 之後喊一聲 math.sqrt(16) 就拿到 4.0，不必知道底層是牛頓法還是查表。這叫程序抽象化。

把同一招用在資料上，就是抽象資料型別（ADT）：只描述資料有哪些操作、行為是什麼， 完全不管怎麼實作。使用者摸到的是外殼，實作藏在殼裡面一層，這層包裝叫封裝。 而 ADT 的具體實作，就是我們整學期要磨的東西：資料結構。

為什麼值得花一學期？ 同一個 ADT 通常有好幾種實作，介面一樣、效能天差地遠。第 2 章開始你會看到： 同一個問題，一種寫法跑一秒，另一種要跑到天荒地老。學會「評估解法」跟學會「寫出解法」一樣重要。
為什麼用 Python？ Python 直譯執行、語法貼近虛擬碼，內建的 list 和 dict 又直接對應課程的核心資料結構： 你可以把力氣花在演算法本身，而不是語言儀式上。課程用 pythonds 這本書，程式碼跟著書的風格走。
PART 01 · 原子型別

原子型別與變數：名字綁到物件上 pythonds §1.7–1.8.1

Python 內建的原子型別主要是 int、float 與 bool； 算術運算子 + - * / % ** // 全員到齊。先看一段講義的示範， 兩個除法運算子的分工要先站穩：

print(2 + 3 * 4) # 14，先乘除後加減 print((2 + 3) * 4) # 20，括號改變順序 print(2 ** 10) # 1024，次方是內建運算子 print(7 / 3) # 2.3333…，/ 永遠是真除法 print(7 // 3) # 2，// 才是整數除法（向下取整） print(7 % 3) # 1，取餘數 print(2 ** 100) # 1267650600228229401496703205376：精確！
›點下面的型別卡，看各型別的個性與地雷。
int 要多大有多大
Python 的整數是任意精度： 2 ** 100 直接算出精確的 31 位數，不會溢位、不會變成近似值。 大整數運算的成本會隨位數成長，但正確性永遠不打折。
兩種除法
7 / 3 是 2.333…（結果一律是 float）； 7 // 3 是 2（向下取整）。想要商和餘數就用 // 搭 %。

布林、比較與邏輯運算子

bool 只有 True 和 False 兩個值，搭配 and、or、not。比較運算的結果也是 bool。整理成一張表：

運算寫法說明
大小比較< > <= >=結果是 bool
相等／不等== 　!=注意 = 是賦值，== 才是比較
邏輯且／或／非and 　or 　not短路求值：左邊定案就不看右邊
連鎖比較1 < 5 < 10合法且符合數學直覺：等於 1 < 5 and 5 < 10
連鎖比較是正字標記 1 < 5 < 10 印出 True：Python 把它展開成 1 < 5 and 5 < 10，跟數學課寫的區間一模一樣。 同理 10 < 5 < 3 是 False，因為第一段就倒了。

變數是「名字」，不是盒子

執行 the_sum = 0 時，Python 先造出整數物件 0，再把名字 the_sum 綁到它身上。之後的賦值都是改綁： 名字轉頭指向新物件，型別跟著新物件走。按按鈕看三步的變化：

›照順序按三顆按鈕，看名字與物件怎麼變。
程式對照 CODE
the_sum = 0 # 名字綁到整數物件 0 the_sum = the_sum + 1 # 算出新物件 1，改綁 the_sum = True # 改綁到 bool 物件：型別跟人走
動態型別的規矩
名字本身沒有型別，物件才有。識別字以字母或底線開頭、大小寫視為不同； 保留字（class、while、def）不能拿來當名字。 同一個名字待會兒指向誰，要看最後一次賦值。

PART 02 · 集合型別

集合型別：list、string、tuple、set、dict pythonds §1.8.2

有序集合（sequence）有 list、string、tuple； 無序集合有 set 和 dict。這一節的幾張表整學期都會回頭查， 值得現在花時間讀熟。

list：什麼都能裝的動態序列

my_list = [1, 3, True, 6.5]：元素可以異質、可以動態增刪， 切片語法 my_list[1:3] 直接取一段（含 1 不含 3）。

range：等差數列產生器

單引號雙引號都行。串接用 +、重複用 *、長度是 len()。重點： 字串是不可變的，my_name[0] = "X" 直接丟 TypeError； 想要「改過的字串」，得造一個新的。

tuple：不可變的 list

my_tuple = (2, True, 4.96)：索引、切片、len、重複都跟 list 一樣， 唯獨不能改：my_tuple[1] = False 丟 TypeError。 需要「保證不被動到」的一組值時用它，之後很多資料會以 tuple 的身分現身（例如 dict 的 items）。

PART 03 · 輸入輸出

輸入與輸出：input、print 與 f-string pythonds §1.9

input(提示字串) 讀進來的永遠是字串： 要當數字用得自己轉，radius = float(input("半徑？"))。 print() 預設用空格分隔、行尾換行，兩者都能改： print("Hello", "World", sep="***", end="")。

格式化字串一路演化：% 舊式 → .format() → f-string（現代首選）。 f-string 的格式規格寫在冒號後面：f"{item:>10}" 靠右塞進寬度 10 的欄位、 f"{price:10.2f}" 寬度 10、小數 2 位。

f"The {item:>10} costs {price:…f} cents"

        
›調整格式規格，馬上看排版結果（item = "banana"、price = 24）。

        

          
          
          
          
        

      

    

    

      

        
三代同堂

        
"%s is %d" % (name, age)
"{} is {}".format(name, age)
f"{name} is {age}"

        
% 是 C 家族傳下來的格式字串；f-string 把運算式直接放進字面值裡，可讀性最好，本課程一律用它。

PART 04 · 控制結構

  
控制結構：迴圈、選擇與串列生成式 pythonds §1.10

  
演算法需要兩種控制結構：迭代與選擇。Python 的程式區塊由縮排決定：
  冒號之後往右縮的那一段就是本體。縮排不是排版建議，是語法本身，混用 tab 和空格是初學第一大坑。


  
兩種迴圈

  

    
寫法用途

    
      
while counter <= 5:條件為真就重複

      
for item in [1, 3, 6, 2, 5]:走訪任何可迭代物件的成員

      
for i in range(5):計數迴圈：搭配 range 控制起點、終點、步幅

    
  

  

word_list = ["cat", "dog", "rabbit"]
letter_list = []
for a_word in word_list:
    for a_letter in a_word:      # 巢狀迴圈：字串也可迭代
        letter_list.append(a_letter)


  
選擇：if、elif、else

  
巢狀 if 疊四層看得眼花，慣用寫法是 elif 一路接平：
  最後的 else 是保底，漏了它，所有條件都不成立時就什麼都不做：

  

if score >= 90:   print("A")
elif score >= 80: print("B")
elif score >= 70: print("C")
elif score >= 60: print("D")
else:             print("F")


  
串列生成式（list comprehension）

  
「取 1 到 10 的奇數、各自平方、收進 list」這類篩選＋轉換需求，Python 一行解決：

  

sq_list = [x * x for x in range(1, 11) if x % 2 != 0]
# [1, 9, 25, 49, 81]：for 走訪、if 篩選、最前面的運算式轉換

  
讀法從中間開始：for 產生候選、if 留下想要的、
  開頭的運算式決定進 list 的長相。跟等價的四行迴圈相比，意圖一眼可見。

PART 05 · 例外處理

  
例外處理：錯誤發生時給程式一條活路 pythonds §1.11

  
錯誤分兩種。語法錯誤是句子寫壞了，直譯器在開始執行前就拒收
  （例如 for i in range(10) 忘了冒號）。邏輯錯誤是程式會跑但答案不對；
  其中最糟的一類會讓程式當場中止，像除以零、用越界的索引，這種執行期錯誤叫例外（exception）。

  

    

      

        

try:
    a_number = int(input("Please enter an integer "))
    print(math.sqrt(a_number))
except:
    print("Bad value for the square root function")
    print("Using the absolute value instead")
    print(math.sqrt(abs(a_number)))  # 補救：改用絕對值

        
›try 圈住可能爆炸的程式碼，except 接住爆炸後決定怎麼辦。沒接住的例外會讓程式當場終止並印出 traceback。

      

    

    

      

        
自己丟例外

        

if a_number < 0:
    raise RuntimeError(
      "You can't use a negative number")
else:
    print(math.sqrt(a_number))

        
與其把負數餵給 sqrt 拿到 ValueError，不如自己檢查、自己 raise，錯誤訊息由你定義。

      

      

        
常見例外快查

        
ZeroDivisionError 除以零、
        IndexError 索引越界、KeyError dict 查無此 key、
        TypeError 型別不對、ValueError 值不合法（如 int("abc")）。

PART 06 · 函式

  
定義函式與參數傳遞 pythonds §1.12

  
函式用 def 定義：名字、參數列、本體，用 return 交回結果
  （沒寫 return 就回 None）。def square(n): return n ** 2
  就是完整的一課：不用宣告型別，呼叫時傳什麼就是什麼。

  
參數傳遞的規矩一句話：傳的是物件的參考，名字是區域的。
  函式裡把參數改綁到新物件，呼叫端不受影響；
  但透過參數就地修改可變物件（例如 list.append），呼叫端看得一清二楚。

  

    

      

        

        
›比較「改綁」與「就地修改」對呼叫端的影響。

        

          
          
          
        

      

    

    

      
程式對照 CODE

def square(n):
    return n ** 2
 
print(square(3))          # 9
print(square(square(3)))  # 81：呼叫可以組合
 
def rebind(x):    x = 99          # 改綁區域名字
def mutate(lst):  lst.append(99)  # 就地修改物件

      

        
兩個實務慣例

        

        ① 函式開頭用三引號寫 docstring，help() 讀得到。

        ② 要改呼叫端的資料，讓意圖明顯：不是回傳新值讓對方接，就是在文件裡講清楚會就地修改。

PART 07 · 類別

  
自訂型別：把 Fraction 一步一步蓋起來 pythonds §1.13.1

  
浮點數存 1/3 只能存個近似值。想要「精確的分數」，就得自己造型別。
  講義用五步把 Fraction 從空殼蓋成能加、能比、能印的完整類別，每一步都有理由。


  
第 1 步：__init__ 建構子與底線慣例

  

class Fraction:
    """Class Fraction"""
    def __init__(self, top, bottom):
        self._num = top      # 單底線：內部狀態的君子協定
        self._den = bottom

  
建構子固定叫 __init__，第一個參數 self 是物件自己，
  成員都掛在 self 身上。底線開頭（_num）是「請勿直接碰」的慣例；
  雙底線（__num）會觸發名稱改編（name mangling），外部存取直接失敗。
  Python 的封裝靠慣例與文件，不靠強制。


  
第 2 步：預設參數（多載的 Python 解法）

  

def __init__(self, top=0, bottom=1):
    ...
# Fraction(3, 5) → 3/5；Fraction(5) → 5/1；Fraction() → 0/1

  
一個建構子配上預設參數，三種呼叫方式一次滿足；
  另一招是 @classmethod 工廠方法（如 Fraction.from_top(5)），
  講義兩種都示範了。使用者根本分不出你選了哪種寫法，這正是封裝的意思。


  
第 3 步：__str__，讓 print 認得你

  
print(my_fraction) 一開始印出 <__main__.Fraction object at 0x...>：
  預設表示法只給型別和位址，等於什麼都沒說。定義 __str__ 之後 print、str()、f-string 全部改口：

  

def __str__(self):
    return f"{self._num}/{self._den}"
# print(my_fraction) → 3/5；f"I ate {my_fraction} of pizza" 也通


  
第 4 步：__add__，加法要順便約分

  
通分相加：a/b + c/d = (ad + cb)/bd。直接這樣存會愈加愈肥（1/4 + 1/2 = 6/8），
  所以加完用最大公因數約分，6/8 出門前變 3/4。這維護了一條不變量：
  Fraction 永遠是最簡分數，其他方法都可以放心依賴這件事。

  

def gcd(m, n):
    while m % n != 0:
        m, n = n, m % n     # 歐幾里得演算法
    return n
 
def __add__(self, other_fraction):
    new_num = self._num * other_fraction._den + \
              self._den * other_fraction._num
    new_den = self._den * other_fraction._den
    common = gcd(new_num, new_den)
    return Fraction(new_num // common, new_den // common)

  
定義了 __add__，f1 + f2 就會自動翻譯成
  f1.__add__(f2)：這就是運算子多載的 Python 版本。


  
第 5 步：__eq__，定義「相等」是什麼意思

  
沒定義 __eq__ 之前，f1 == f2 比的是「是不是同一個物件」（身分）。

對應程式：g1 = AndGate("g1")、g3 = OrGate("g3")、g4 = NotGate("g4")
c1 = Connector(g1, g3)、c3 = Connector(g3, g4)，然後 g4.get_output()。

      

      

        這段設計裡的四個關鍵機制
        動態繫結：get_output() 寫在父類，呼叫 self.perform_gate_logic() 時，
        執行期照物件的實際類別找方法：AndGate 的物件就跑 AndGate 的版本。

        缺席即抽象：LogicGate 沒定義 perform_gate_logic，直接拿它造物件再呼叫就 AttributeError：
        它天生只適合當基底。

        super().__init__(lbl)：呼叫父類建構子，把共同的初始化交回去。

        HAS-A：Connector 不繼承 LogicGate，它「擁有」兩個閘（from_gate、to_gate）。
        IS-A 用繼承、HAS-A 用成員，分清楚這兩種關係是物件導向設計的第一堂課。
      

    

    

      
基底與 BinaryGate CODE

class LogicGate:
    def __init__(self, lbl):
        self._label = lbl
        self._output = None
    def get_output(self):
        self._output = self.perform_gate_logic()
        return self._output   # 邏輯由子類提供
 
class BinaryGate(LogicGate):   # 括號＝繼承
    def __init__(self, lbl):
        super().__init__(lbl)   # 呼叫父類建構子
        self._pin_a = None
        self._pin_b = None
    def set_from_pin(self, source):
        if self._pin_a is None:   self._pin_a = source
        elif self._pin_b is None: self._pin_b = source
        else: raise RuntimeError("NO EMPTY PINS")

      
具體的閘 CODE

class AndGate(BinaryGate):
    def __init__(self, lbl):
        super().__init__(lbl)
    def perform_gate_logic(self):
        a = self.get_pin_a()
        b = self.get_pin_b()
        return 1 if a == 1 and b == 1 else 0
# OrGate、NotGate 同款，只換這一個方法

      
Connector CODE

class Connector:      # 不在閘的階層裡：HAS-A
    def __init__(self, fgate, tgate):
        self.from_gate = fgate
        self.to_gate = tgate
        tgate.set_from_pin(self)  # 把自己插上目的閘的腳位
    def get_from(self):
        return self.from_gate

抽象與封裝

      ADT：只講操作與行為的資料模型。
資料結構：ADT 的具體實作。

      封裝：把狀態藏起來，只留公開介面。
介面：元件之間交換資訊的邊界。

    
語言機制

      直譯器：逐行執行原始碼，錯誤多半到執行期才現形。

      動態型別：型別跟著物件走，名字只是綁定。

      可變／不可變：list、dict 可變；string、tuple 不可變。

      縮排區塊：縮排就是語法。

    
錯誤三兄弟

      語法錯誤：執行前被擋。
邏輯錯誤：會跑但答案錯。

      例外：執行期被打斷，可 try/except 接住、可 raise 自己丟。

    
物件導向

      __init__：建構子，self 是物件自己。
魔術方法：__str__、__add__、__eq__ 讓內建語法認得你。

      覆寫：子類換掉父類的實作。
多型：同一介面、各自表述（動態繫結）。

      深／淺相等：比值 vs 比身分。
